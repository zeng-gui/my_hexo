# 博客文章规范

## Front Matter 必填项

```yaml
---
title: 文章标题
date: YYYY-MM-DD HH:MM:SS
tags:
  - 标签1
  - 标签2
categories:
  - 分类          # 一级分类
  # - [分类, 子分类]  # 多级分类
toc: true
comments: true
---
```

## 摘要截断

**必须**在文章开头适当位置插入 `<!-- more -->` 标签。否则首页会显示全文。

位置建议：第一段介绍性文字之后、正文第一个二级标题之前。

## 文件名规范

- 小写英文 + 短横线：`my-post-title.md`
- 不要用中文文件名、不要用冒号等特殊字符
- Hexo 生成的默认 slug 可能包含中文，需要手动重命名

## 写作规范

- 中英文之间加空格：`使用 Hexo 搭建` 而不是 `使用Hexo搭建`
- 代码块必须标注语言：` ```python ` 而不是 ` ``` `
- 标题层级从 `##` 开始（`#` 留给文章标题）
- 专有名词保持官方大小写：GitHub、JavaScript、Python

## 分类体系（一级分类，不超过 7 个）

- 技术笔记 — 语言、框架、工具的使用记录
- 实战项目 — 完整项目从 0 到 1
- 环境搭建 — 开发环境、服务器、CI/CD
- 问题排查 — 踩坑记录、报错解决
- 学习资源 — 书单、课程、文档整理
- 随笔 — 非技术类

## 代码块中的特殊字符

如果代码块内容包含 YAML 解析冲突的字符（如冒号后跟特殊内容），用 `<!-- raw -->` 包裹或改用缩进代码块。
